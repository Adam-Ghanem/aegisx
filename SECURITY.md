# Security Policy

Do not open public issues for suspected vulnerabilities. Report them through GitHub's private vulnerability reporting feature when enabled, including affected revision, impact, reproduction steps using synthetic data, and suggested mitigation. Do not include real credentials, customer data, or offensive payloads.

Phase 1 is a development foundation and is not approved for production or customer telemetry. Supported versions will be published when the first release is made. Maintainers triage reports by tenant-isolation, authentication, authorization, confidentiality, integrity, and availability impact.
